Documenation and Instructions
---

<p>Diese L&ouml;sung stellt f&uuml;r die Entwicklung von AE-Objekten ein Standard f&uuml;r die Planung und f&uuml;r das Deployment von Releases zur Verf&uuml;gung. Dieser Standard lehnt sich an Prinzipien der Software-Entwicklung an. In Kombination mit der optionalen L&ouml;sung best4Automic Git-Integration kann der Entwickler auf Funktionalit&auml;en eines Versionskontrollsystems zur&uuml;ckgreifen.&nbsp;</p>
<p>Mit der b4A Deployment-Solution stehen folgende Services zur Verf&uuml;gung:</p>
<ul>
<li>&Uuml;berpr&uuml;fung von Abh&auml;ngigkeiten und Richtlinien (b4A ComplianceCheck)</li>
<li>Package-Initialisierung</li>
<li>Package-Erstellung</li>
<li>Installation von Packages inklusive Post-Deployment-Arbeiten</li>
<li>Registrierung von Actions</li>
<li>Registrierung von Vorlagen (Templates)</li>
</ul>
<p>Die best4Automic Deployment-Solution basiert auf b4A Packages, die sich an die Struktur der Automic Action Packs anlehnen k&ouml;nnen. In der Standard-Vorkonfiguration wird die Struktur der Action Packs ber&uuml;cksichtigt, so dass Kunden, die bereits Action Packs nutzen, sofort starten k&ouml;nnen.</p>
